1. VISÃO GERAL — O QUE VAMOS CONSTRUIR

Este guia ensina a criar um agente de IA completo para WhatsApp que funciona como atendente virtual 24/7. O agente recebe mensagens, entende a intenção do cliente usando IA (Llama 3.3 via Groq), consulta dados, responde automaticamente e pode executar ações — tudo sem escrever código.

O que o agente faz:

  1. Recebe mensagens de qualquer pessoa no WhatsApp
  2. Entende a intenção usando IA (pergunta, compra, suporte, saudação)
  3. Responde automaticamente com base no contexto
  4. Consulta banco de dados (pedidos, agendamentos, produtos)
  5. Encaminha para humano quando não souber responder
  6. Envia mensagens proativas (lembretes, promoções)

Casos de uso:

NegócioUso do Agente
Clínica médicaAgendar/consultar/cancelar consultas
RestauranteCardápio, pedidos, horários
E-commerceStatus de pedido, trocas, dúvidas
AdvocaciaTriagem de clientes, agendamento
ImobiliáriaImóveis disponíveis, visitas
Salão de belezaAgendamento, serviços, preços
💡 Por que esta stack?
• n8n — Open-source, visual, sem código. 400+ integrações.
• Evolution API — Open-source, conecta ao WhatsApp via QR Code. Sem depender da API oficial cara da Meta.
• Groq — API de IA gratuita com Llama 3.3 70B. Muito rápida.
Custo total: R$ 0 (só o servidor ~R$ 60/mês)

4. PASSO 1: CONFIGURAR O SERVIDOR VPS

4.1 — Conectar ao servidor via SSH

ssh root@IP_DO_SERVIDOR

4.2 — Atualizar o sistema

apt update && apt upgrade -y

4.3 — Instalar Docker e Docker Compose

# Instalar Docker curl -fsSL https://get.docker.com | sh # Habilitar Docker no boot systemctl enable docker systemctl start docker # Instalar Docker Compose apt install docker-compose-plugin -y # Verificar docker --version docker compose version

4.4 — Configurar Firewall (UFW)

ufw allow 22/tcp # SSH ufw allow 80/tcp # HTTP ufw allow 443/tcp # HTTPS ufw allow 5678/tcp # n8n ufw allow 8080/tcp # Evolution API ufw enable

5. PASSO 2: INSTALAR O N8N

5.1 — Criar diretório e docker-compose.yml

mkdir -p /opt/n8n && cd /opt/n8n cat > docker-compose.yml << 'EOF' version: '3.8' services: n8n: image: n8nio/n8n:latest container_name: n8n restart: always ports: - "5678:5678" environment: - N8N_HOST=n8n.seudominio.com - N8N_PORT=5678 - N8N_PROTOCOL=https - WEBHOOK_URL=https://n8n.seudominio.com/ - GENERIC_TIMEZONE=America/Sao_Paulo - TZ=America/Sao_Paulo - N8N_SECURE_COOKIE=false volumes: - n8n_data:/home/node/.n8n volumes: n8n_data: EOF

5.2 — Subir o n8n

docker compose up -d # Verificar se está rodando docker logs n8n --tail 20

5.3 — Acessar a interface

Acesse: http://IP_DO_SERVIDOR:5678
Crie a conta de administrador na primeira vez.

6. PASSO 3: INSTALAR EVOLUTION API

6.1 — Criar diretório e docker-compose.yml

mkdir -p /opt/evolution && cd /opt/evolution

6.2 — Criar o docker-compose.yml completo

cat > docker-compose.yml << 'EOF' version: '3.8' services: postgres: image: postgres:15 container_name: evolution-db restart: always environment: POSTGRES_USER: evolution POSTGRES_PASSWORD: evolution2025 POSTGRES_DB: evolution volumes: - pg_data:/var/lib/postgresql/data ports: - "5432:5432" redis: image: redis:7-alpine container_name: evolution-redis restart: always command: redis-server --appendonly yes volumes: - redis_data:/data evolution-api: image: atendai/evolution-api:latest container_name: evolution-api restart: always depends_on: - postgres - redis ports: - "8080:8080" environment: - SERVER_TYPE=http - SERVER_PORT=8080 - CORS_ORIGIN=* - CORS_METHODS=POST,GET,PUT,DELETE - CORS_CREDENTIALS=true - DEL_INSTANCE=false - DATABASE_ENABLED=true - DATABASE_PROVIDER=postgresql - DATABASE_CONNECTION_URI=postgresql://evolution:evolution2025@postgres:5432/evolution - REDIS_ENABLED=true - REDIS_URI=redis://redis:6379 - AUTHENTICATION_TYPE=apikey - AUTHENTICATION_API_KEY=sua-chave-secreta-aqui-mude-isso - WEBHOOK_GLOBAL_ENABLED=false - WEBHOOK_EVENTS_MESSAGES_UPSERT=true - WEBHOOK_EVENTS_MESSAGES_UPDATE=true volumes: - evolution_data:/evolution/store volumes: pg_data: redis_data: evolution_data: EOF
🔑 IMPORTANTE: Altere AUTHENTICATION_API_KEY para uma chave segura sua!
Exemplo: minha-chave-super-secreta-2026
Guarde essa chave — você vai precisar dela para conectar ao n8n.

6.3 — Subir a Evolution API

docker compose up -d # Verificar logs docker logs evolution-api --tail 30
Aguarde até ver: "Server is running on port 8080" nos logs.

7. PASSO 4: CONECTAR O WHATSAPP (QR CODE)

7.1 — Acessar o Manager da Evolution API

http://IP_DO_SERVIDOR:8080/manager

7.2 — Criar uma instância

  1. Abra o Postman ou use o curl diretamente
  2. Crie a instância com o seguinte comando:
curl -X POST http://IP_DO_SERVIDOR:8080/instance/create \ -H "Content-Type: application/json" \ -H "apikey: sua-chave-secreta-aqui-mude-isso" \ -d '{ "instanceName": "meu-agente", "token": "", "number": "", "qrcode": true, "integration": "WHATSAPP-BAILEYS", "reject_call": true, "msg_call": "", "groups_ignore": true, "always_online": true, "read_messages": true, "read_status": true }'

7.3 — Ler o QR Code

# Gerar/obter o QR Code curl -X GET "http://IP:8080/instance/connect/meu-agente" \ -H "apikey: SUA_CHAVE"

A resposta conterá uma imagem em base64. Você pode:

  • Salvar como imagem e escanear direto
  • Usar o Postman com a interface visual
  • Acessar no browser: http://IP:8080/manager

7.4 — Escanear com o WhatsApp

  1. Abra o WhatsApp no telefone com o chip dedicado
  2. Vá em Configurações → Aparelhos conectados
  3. Toque em Conectar um aparelho
  4. Escaneie o QR Code gerado pela Evolution API
  5. Aguarde a conexão (pode levar 30-60 segundos)

7.5 — Verificar conexão

curl -X GET "http://IP:8080/instance/connectionState/meu-agente" \ -H "apikey: SUA_CHAVE"
✅ Sucesso: A resposta deve conter "state": "open"
🔄 Conectando: "state": "connecting" — aguarde e tente novamente
❌ Falha: Gere um novo QR Code repetindo o passo 7.3

8. PASSO 5: CONFIGURAR GROQ API (IA GRATUITA)

8.1 — Criar conta no Groq

  1. Acesse: console.groq.com
  2. Clique em "Sign Up" — pode logar com GitHub ou email
  3. Confirme o email

8.2 — Gerar API Key

  1. No painel do Groq, vá em "API Keys" (menu lateral)
  2. Clique em "Create API Key"
  3. Dê um nome: "n8n-agente-whatsapp"
  4. Copie a key (começa com gsk_...)

8.3 — Escolher o modelo

ModeloVelocidadeQualidadeGratuito
llama-3.3-70b-versatile⚡⚡⚡ Muito rápida⭐⭐⭐⭐ Excelente✅ Sim
llama-3.1-8b-instant⚡⚡⚡⚡ Ultra rápida⭐⭐⭐ Boa✅ Sim

gemma2-9b-it⚡⚡⚡ Rápida⭐⭐⭐ Boa✅ Sim
Recomendação: Use llama-3.3-70b-versatile — melhor custo/benefício. Muito rápido e inteligente para atendimento.

8.4 — Configurar credencial no n8n

  1. No n8n, vá em "Settings" (engrenagem) → "Credentials"
  2. Clique em "Add Credential"
  3. Pesquise por "Groq" ou "OpenAI"
  4. Cole a API key do Groq
  5. Se usar "OpenAI", ajuste a base URL para: https://api.groq.com/openai/v1
  6. Salve com nome: "Groq - Agente WhatsApp"

10. PASSO 7: CONECTAR EVOLUTION → N8N

10.1 — Configurar webhook no n8n

No mesmo nó Webhook (Passo 9.2), copie a URL de Produção.

10.2 — Registrar webhook na Evolution API

curl -X PUT http://IP_DO_SERVIDOR:8080/webhook/set/meu-agente \ -H "Content-Type: application/json" \ -H "apikey: SUA_CHAVE_EVOLUTION" \ -d '{ "url": "https://n8n.seudominio.com/webhook/whatsapp-agente", "webhook_by_server": true, "webhook_base64": false, "events": { "messages.upsert": true, "messages.update": false, "connection.update": true } }'

10.3 — Verificar se o webhook está ativo

curl -X GET http://IP_DO_SERVIDOR:8080/webhook/find/meu-agente \ -H "apikey: SUA_CHAVE_EVOLUTION"

A resposta deve mostrar a URL configurada.

14. DICAS AVANÇADAS E BOAS PRÁTICAS

  1. 1. Human-in-the-loop: Configure um nó "Switch" que mensagens complexas para atendimento humano via Telegram ou email.
  2. 2. Memória de contexto: Use PostgreSQL no n8n para guardar histórico de conversa e manter contexto entre mensagens.
  3. 3. Rate limiting: Adicione um nó "IF" limite mensagens por minuto por número para evitar spam.
  4. 4. Horário somente expediente: Adicione nó que após o expediente envie: "Estamos fora do horário. Retornaremos amanhã!"
  5. 5. Mensagens em massa: Para disparos marketing, use a Evolution API diretamente com delays entre envios.
  6. 6. Monitoramento: Configure alertas no n8n (email/Telegram) quando workflow falhar > 3 vezes.
  7. 7. Grupos: A Evolution API ignora grupos por default (groups=_ignore: true). Pode mudar se quiser moderar grupos.
  8. 8. Mídia: O suporta envio/recebimento de imagens, áudio, documentos e localização.
  9. 9. Backup: Faça backup do volume n8n_data regularmente: docker run --rm -v n8n_data:/data -v $(pwd):/backup tar czf /backup/n8n-backup.tar.gz /data
  10. 10. Atualização: docker compose pull && docker compose up -d para atualizar Evolution API e n8n.
